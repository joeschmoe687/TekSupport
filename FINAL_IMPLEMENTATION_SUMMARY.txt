                          KEY FEATURES
═══════════════════════════════════════════════════════════════════════════

🤖 REAL-TIME AI DIAGNOSTICS
   • Analyzes readings every second
   • Color-coded alerts: 🟢 Normal, 🟡 Warning, 🔴 Critical
   • Contextual messages: "Suction pressure 95 PSI is LOW..."
   • Top 3 possible causes per alert

🧠 PATTERN RECOGNITION
   • Low Charge Detection (low P + high SH + low SC)
   • Overcharge Detection (high P + low SH + high SC)
   • TXV Flooding (high suction + low SH)
   • Restricted Metering Device (low suction + high SH)

🔧 SMART TROUBLESHOOTING
   • Beginner Mode: Detailed tooltips & explanations
   • Expert Mode: Concise bullet points
   • Dynamic paths based on detected issues
   • Checkable steps for progress tracking
   • Safety cautions highlighted

📊 ML DATA PIPELINE
   • One-tap "Capture Reading" button
   • Automatic anonymization (no customer PII)
   • Privacy toggle in Settings
   • Firebase collection: ml_hvac_readings
   • Batch upload on job completion

🌡️ SYSTEM SUPPORT

═══════════════════════════════════════════════════════════════════════════
                      HOW IT WORKS (USER FLOW)
═══════════════════════════════════════════════════════════════════════════

1. Technician opens Gauge screen
2. Bluetooth sensors connect and stream readings
3. ✨ DIAGNOSTIC CARD APPEARS showing status
   • Green = All readings normal
   • Yellow = Needs attention  
   • Red = Critical issue detected
4. Tap "FIX" button → Troubleshooting guide opens
5. Follow step-by-step instructions
6. Tap "CAPTURE" to save reading for ML training

═══════════════════════════════════════════════════════════════════════════
                       TECHNICAL ARCHITECTURE
═══════════════════════════════════════════════════════════════════════════

DATA FLOW:
Bluetooth Sensors
    ↓
DeviceDataService (real-time readings)
    ↓
GaugeScreen (stores & displays)
    ↓
_updateCalculations() (superheat/subcool)
    ↓
DiagnosticEngine.analyze() (AI analysis)
    ↓
DiagnosticResult (alerts + status)
    ↓
DiagnosticCard (UI display)
    ↓
TroubleshootingSheet (step-by-step guidance)

ML COLLECTION:
GaugeScreen "Capture" Button
    ↓
MLDataService.captureReading()
    ↓
HvacReading Model (anonymized)
    ↓
Firebase ml_hvac_readings Collection

═══════════════════════════════════════════════════════════════════════════
                      FIREBASE STRUCTURE
═══════════════════════════════════════════════════════════════════════════

Collection: ml_hvac_readings

Document Example:
{
  "jobId": "optional_job_id",
  "technicianId": "anonymized_or_null",
  "timestamp": "2024-12-21T10:30:00Z",
  "systemType": "airConditioning",
  "refrigerant": "r410a",
  "suctionPressure": 120.5,
  "dischargePressure": 380.2,
  "superheat": 12.5,
  "subcool": 10.2,
  "outcome": "pass",
  "isAnonymized": true
}

Security Rules:
• Read: Authenticated techs/admins only
• Write: Authenticated users with isAnonymized=true
• No customer PII allowed

═══════════════════════════════════════════════════════════════════════════
                       PRIVACY & SECURITY
═══════════════════════════════════════════════════════════════════════════

✅ WHAT GETS STORED:
   • Technical readings (pressures, temps, calculations)
   • System type & refrigerant
   • Equipment model (from nameplate)
   • Timestamp & adjustments made

❌ WHAT NEVER GETS STORED:
   • Customer names
   • Customer addresses  
   • Job locations (GPS)
   • Phone numbers
   • Any PII

🔒 USER CONTROL:
   • Settings → "Share Diagnostic Data for ML"
   • Toggle on/off anytime
   • Preference persists
   • When disabled: readings saved locally only
